# Duration Parsing

Two small helpers for dealing with duration inputs. Useful any time you're taking a timeout or mute duration from a user.

## parseDuration

Converts a duration string into milliseconds.

```js
parseDuration('30s')  // 30000
parseDuration('10m')  // 600000
parseDuration('2h')   // 7200000
parseDuration('7d')   // 604800000
```

Supported suffixes: `s` (seconds), `m` (minutes), `h` (hours), `d` (days).

Returns `NaN` for invalid input, so check before using:

```js
const { parseDuration, errorEmbed } = require('discord.js-toolbox');

const ms = parseDuration(input);
if (isNaN(ms)) {
  return interaction.reply({
    embeds: [errorEmbed('Invalid duration. Use formats like `10m`, `2h`, or `7d`.')],
    ephemeral: true,
  });
}
```

---

## formatDuration

Converts milliseconds back into a human-readable string.

```js
formatDuration(30000)    // "30 seconds"
formatDuration(600000)   // "10 minutes"
formatDuration(3600000)  // "1 hour"
formatDuration(86400000) // "1 day"
```

Useful for confirmation messages after applying a timeout:

```js
const { safeTimeout, parseDuration, formatDuration, successEmbed } = require('discord.js-toolbox');

const ms = parseDuration('2h');
await safeTimeout(member, ms, reason);
await interaction.reply({
  embeds: [successEmbed(`**${member.user.tag}** was timed out for ${formatDuration(ms)}.`)],
});
```
